# Системный дизайн проезда SFMShop

SFMShop - микросервисное веб-приложение с использованием FastAPI.


## Архитектура:
- **API-сервисы** — обрабатывают HTTP-запросы, бизнес-логику и работу с БД
- **PostgreSQL** — хранение основной информации (товары, пользователи, заказы)
- **Redis** — кэширование данных для ускорения чтения часто запрашиваемых ресурсов
- **Swagger / OpenAPI** — документация и тестирование API.


## Масштабирование
### Горизонтальное масштабирование API
- API-сервисы можно масштабировать по количеству RPS или CPU/latency
- При росте нагрузки добавлять новые экземпляры через балансировщик

### Вертикальное масштабирование БД
- Дать основной БД (PostgreSQL) больше CPU и RAM для обработки записей
- Для увеличения чтения (GET-запросов) создать реплики и использовать кэш Redis


## Кэширование
- Redis используется для кэширования списка товаров (products)
- Кэш имеет TTL 1 час, после чего данные обновляются.
- Если Redis недоступен — API обращается напрямую к БД, чтобы не блокировать пользователей
- В браузере кэшируются статические ресурсы


## Метрики производительности

- **RPS**: 5000+  
- **Latency**: < 100 мс для основных запросов  
- **Availability**: > 99.9%
- **Мониторинг**:
  - Нагрузка на CPU и RAM
  - Количество запросов к БД и Redis
  - Ошибки HTTP 4xx / 5xx
