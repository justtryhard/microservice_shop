# Архитектура асинхронной обработки заказов для проекта SFMShop

## Задачи для очереди
1. Отправка email-уведомлений  
    - пользователь уведомляется о создании заказа и изменении его статуса
    - прочие рассылки
2. Обновление склада
    - изменение количества товаров после заказа или поставки
    - проверка наличия товара
3. Генерация отчётов
    - отчёты по проданным товарам
    - статистика по юзерам
    - анализ заказов
4. Другие фоновые задачи
    - логирование
    - интеграции с внешними API
    - кэширование данных

## Надёжность
1. Гарантии доставки сообщений 
    - сообщения сохраняются в очереди до обработки
    - сообщения не удаляются, пока не будет подтверждения обработки
    - сообщение будет обработано хотя бы 1 раз
2. Retry-логика при ошибках
    - при ошибке задача возвращается в очередь
    - количество повторных попыток ограничено (3-5)
    - экспоненциальные задержки между ретраями
3. Обработка ошибок
    - ошибки логируются
    - при превышении лимита ретраев сообщение отправляется в отдельную очередь (DLQ)

## Масштабирование
1. Горизонтальное масштабирование Consumer
   - запуск нескольких инстансов Consumer
   - каждый инстанс обрабатывает задачи параллельно
2. Балансировка нагрузки между воркерами
   - 1 Producer - много Consumer: задачи автоматически распределяются между Consumer
3. Мониторинг очередей
   - отслеживание времени обработки задач, ошибок, количества ретраев
   - уведомления при росте очереди, увеличении кол-ва ошибок, отвалах Consumer
