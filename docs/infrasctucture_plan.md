# Документация по инфраструктуре

## 1. Архитектура сервисов

Проект разделён на несколько сервисов, чтобы обеспечить масштабируемость и изоляцию компонентов:

1. APP - основное приложение (API, бизнес-логика)
2. Redis - кэширование
3. PostgreSQL - база данных для хранения пользователей, продуктов и заказов (в разработке)

## 2. Взаимодействие сервисов
APP -- PostgreSQL: подключение через переменную окружения в .env  
APP -- Redis: подключение через переменную окружения в .env  

## 3. CI/CD
Используется GitHub Actions для автоматизации:  
При каждом push или pull-request происходит запуск тестов и проверка покрытия.  
Возможен автоматический деплоя при мердже в main.  

## 4. Рекомендации по масштабированию:
APP:
1. Вертикальное
- увеличение CPU, RAM
2. Горизонтальное
- запуск нескольких инстансов
- балансировка нагрузки

БД:
- Выделить основной БД (PostgreSQL) больше CPU и RAM для обработки записей
- Для увеличения скорости чтения создать реплики GET-запросов и использовать кэш Redis
