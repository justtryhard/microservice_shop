# Масштабирование обработки задач через RabbitMQ

## Архитектура
- Producer отправляет задачи в очередь при создании заказа
- Consumer слушает очередь и выполняет задачи

## Балансировка нагрузки
prefetch_count обеспечивает балансировку нагрузки между несколькими воркерами: брокер отдаёт по одному сообщению каждому consumer только после ack-подтверждения.

## Запуск нескольких воркеров
Несколько инстансов consumer можно запускать следующим образом:
python src/servicer/queue_consumer.py &
python src/servicer/queue_consumer.py &
python src/servicer/queue_consumer.py &

rabbitmq будет распределять задачи между ними равномерно

## Надёжность
каждое сообщение подтверждается basic_ack
если задача не выполнена, отрабатывается retry-логика (3 попытки с экспоненциальной задержкой)

## Масштабирование
Горизонтальное - добавление новых воркеров без изменения проекта
