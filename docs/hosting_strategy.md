# Стратегия хостинга для проекта SFMShop

## Оценка нагрузки
Текущая нагрузка: < 10 RPS
Ожидаемая нагрузка: 20-50 RPS
Пиковая нагрузка: < 100 RPS

## Выбор хостинга
Выбранный хостинг: VPS 
Обоснование: Невысокие ожидаемые нагрузки, VPS дешевле, PaaS и клауды будут излишни.
+ VPS легко масштабировать вертикально и он не сложен в настройке.

## Стратегия масштабирования
### 1. Вертикальное (в первую очередь)
- увеличение CPU, RAM

### 2. Горизонтальное (когда упёрлись в потолок на первом этапе)
- запуск нескольких инстансов
- балансировка нагрузки

## Мониторинг
- логи докера
- отправка алертов в тг 
- prometheus / grafana (в перспективе)

Метрики для мониторинга: RPS, latency, ошибки, нагрузка на CPU и RAM
