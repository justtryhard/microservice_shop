# Подход к тестированию в проекте SFMShop

## Обзор
В проекте реализовано тестирование, включающее
- тестирование моделей
- тестирование эндпойнтов
- тестирование отдельных функций

Целью тестирования являются проверка бизнес-логики и обеспечение стабильности приложения.


## Типы тестов
1. Unit-тесты (создание объектов, проверка валидности, работа методов)
2. Integration-тесты для API (эндпойнты, статусы ответа, корректность данных, работа авторизации)
3. Тесты отдельных функций (функции расчётов из calculations.py)

## Покрытие кода
Для анализа использовался pytest cov
Покрытие моделей: 84%
Покрытие утилит: 88%
Покрытие API: 69%
Общее покрытие: 53%

## Организация тестов
Тесты находятся в директории tests и разбиты на 3 файла в зависимости от назначения.

## Использованные техники
- Фикстуры для создания тестовых моделей и переиспользования данных между тестами
- Моки для изоляции БД, внешних сервисов (Redis, платёжная система)
- Параметризация для тестирования различных сценариев без дублирования кода
- TDD использовался для разработки функций расчёта (calculations.py)
